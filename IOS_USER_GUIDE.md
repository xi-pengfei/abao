# iPhone 使用指南

这份指南给只使用手机的人看。你不需要安装代码，也不需要懂服务器；只需要拿到两个信息：

```text
Server URL
Access Token
```

示例：

```text
Server URL: http://106.52.223.131:8020
Access Token: xxxxxxxxxxxxxxxxx
```

## 1. 用 Safari 打开

请使用 iPhone 自带的 **Safari** 浏览器打开 `Server URL`。

不要优先用微信内置浏览器、QQ 浏览器或其他 App 内置浏览器，因为它们可能不能正常添加到主屏幕。

打开后你会看到聊天页面。

## 2. 填写连接信息

点页面右上角的设置按钮。

填写：

```text
Server URL: 对方给你的 Server URL
Access Token: 对方给你的 Access Token
```

注意：

- `Access Token` 只填 token 本身。
- 不要在前面加 `Bearer`。
- 不要多复制空格或换行。

填完后点保存或测试连接。

如果状态显示已连接，就可以聊天了。

## 3. 添加到手机桌面

在 Safari 页面底部点击 **分享按钮**。

分享按钮通常长这样：

```text
一个方框，里面有向上的箭头
```

然后在菜单里找到：

```text
添加到主屏幕
```

如果没看到，向下滑动菜单，或者点“编辑操作”查找。

进入确认页面后，点右上角：

```text
添加
```

添加完成后，手机桌面上会出现一个图标。以后直接点这个图标使用即可。

## 4. 以后怎么使用

以后从桌面图标打开。

正常情况下，之前保存过的 `Server URL` 和 `Access Token` 会自动保留，不需要每次重新填写。

如果换手机、清理 Safari 数据、或重新添加桌面图标，可能需要重新填写一次。

## 5. 常见问题

### 页面打不开

先确认 `Server URL` 是否完整，例如：

```text
http://106.52.223.131:8020
```

不要漏掉：

```text
http://
```

也不要把中文冒号、空格复制进去。

### 显示未连接

可能原因：

- `Server URL` 填错。
- 服务器暂时没启动。
- 手机网络不稳定。
- 如果使用公司/校园网络，可能屏蔽了某些端口。

可以切换到 4G/5G 再试一次。

### 能打开页面，但发消息失败

优先检查 `Access Token`。

常见错误：

```text
Bearer xxxxx
```

这里是错的。应该只填：

```text
xxxxx
```

### 找不到“添加到主屏幕”

请确认你是在 Safari 里打开页面。

如果你是在微信、QQ、飞书等 App 里打开的，先点右上角菜单，选择：

```text
在 Safari 中打开
```

然后再添加到主屏幕。

### 添加到桌面后图标或名字不对

先删除桌面上的旧图标，再用 Safari 重新打开 `Server URL`，重新添加一次。

如果还是不对，可能是手机缓存了旧信息，过一会儿再试。

## 6. 安全提醒

`Access Token` 相当于这套聊天入口的钥匙，不要发到公开群里，也不要截图公开分享。

如果 token 泄露，需要让部署者在服务器 `.env` 里更换 `ABAO_OWNER_TOKEN`，然后重启服务。
